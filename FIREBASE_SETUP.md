# Подключение Firebase к Pulse

Подробный гайд: от создания проекта Firebase до запуска приложения с реальным чатом между устройствами.

**Что получишь:** регистрация по email/паролю, реальную отправку сообщений между устройствами в реальном времени, синхронизацию между всеми устройствами одного пользователя.

---

## Шаг 1. Создание проекта Firebase (5 минут)

1. Открой https://console.firebase.google.com
2. Войди в Google-аккаунт (любой обычный, отдельный не нужен)
3. Нажми **«Создать проект»** (Create a project)
4. Имя проекта: `pulse-chat` (или любое своё)
5. На вопрос про Google Analytics — можно отключить (не нужно для чата)
6. Жми **«Создать проект»** и подожди 30 секунд

---

## Шаг 2. Включить Authentication (2 минуты)

1. В левом меню → **Build → Authentication**
2. Нажми **«Get started»**
3. Вкладка **Sign-in method**
4. Найди **Email/Password** → нажми → переключи **Enable** → **Save**

Готово, теперь можно регистрировать пользователей.

---

## Шаг 3. Создать Firestore Database (3 минуты)

1. В левом меню → **Build → Firestore Database**
2. Нажми **«Create database»**
3. Выбери регион — для России лучше **`europe-west` (Бельгия)** или **`europe-central2` (Варшава)**, для других стран — ближайший
4. **«Start in test mode»** (потом сменим)
5. Нажми **«Enable»**

⚠️ **Важно:** test mode даёт всем доступ ко всем данным на 30 дней. Перед публикацией нужно поставить правила безопасности (см. ниже).

---

## Шаг 4. Получить ключи приложения (2 минуты)

1. Сверху слева, нажми на **шестерёнку** рядом с «Project Overview»
2. **«Project settings»**
3. Прокрути до **«Your apps»**
4. Нажми иконку **`</>`** (Web)
5. Имя: `pulse-web` (любое)
6. **«Register app»** (галочку про hosting не ставь)
7. Появится код с объектом `firebaseConfig` — это твои ключи

Будет выглядеть так:

```javascript
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "pulse-chat-xxxxx.firebaseapp.com",
  projectId: "pulse-chat-xxxxx",
  storageBucket: "pulse-chat-xxxxx.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef..."
};
```

**Скопируй эти 6 значений.**

---

## Шаг 5. Вставить ключи в проект (1 минута)

В скачанном проекте открой файл `src/firebase/config.js` и замени плейсхолдеры на свои значения:

```javascript
const firebaseConfig = {
  apiKey: 'AIzaSy...',                              // вставь сюда
  authDomain: 'pulse-chat-xxxxx.firebaseapp.com',   // вставь сюда
  projectId: 'pulse-chat-xxxxx',                    // вставь сюда
  storageBucket: 'pulse-chat-xxxxx.appspot.com',    // вставь сюда
  messagingSenderId: '1234567890',                  // вставь сюда
  appId: '1:1234567890:web:abcdef...',              // вставь сюда
};
```

**Сохрани файл.** Всё, проект подключён к Firebase.

---

## Шаг 6. Установить зависимости и запустить

```bash
cd pulse-app
npm install
npx expo start
```

Сканируй QR в Expo Go — приложение откроется. Увидишь экран **«Войди в аккаунт»**.

---

## Шаг 7. Проверка реальной работы

Чтобы проверить что чат правда работает между устройствами:

1. Зарегистрируй **первый аккаунт** на телефоне (например `anna@test.com` / `123456`)
2. Зарегистрируй **второй аккаунт** на компьютере (через Expo Go в браузере или второй телефон) — например `petr@test.com`
3. На втором аккаунте → нажми **«+»** в правом верхнем углу → выбери первого пользователя → откроется чат
4. Напиши сообщение
5. На первом аккаунте сообщение появится **мгновенно** в списке чатов и в самом чате

Так работает live-синхронизация Firestore — каждое устройство подписано на изменения и получает их сразу.

---

## Шаг 8. Безопасность Firestore (важно перед публикацией)

В test mode всем разрешено читать-писать любые данные — это опасно.

1. Firebase Console → **Firestore Database → Rules**
2. Замени содержимое на это:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Пользователи: каждый видит всех, но менять может только свой профиль
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
    }

    // Чаты: только участники могут читать и писать
    match /chats/{chatId} {
      allow read, write: if request.auth != null
        && request.auth.uid in resource.data.members;
      allow create: if request.auth != null
        && request.auth.uid in request.resource.data.members;

      // Сообщения внутри чата
      match /messages/{messageId} {
        allow read, write: if request.auth != null
          && request.auth.uid in get(/databases/$(database)/documents/chats/$(chatId)).data.members;
      }
    }
  }
}
```

3. Нажми **«Publish»**

Теперь только авторизованные участники чата могут читать сообщения.

## Что ещё локальное (нужны отдельные шаги)

⚠️ **Звонки** — экран есть, но это пока интерфейс. Реальные звонки нужно делать через **LiveKit** или **Agora** (отдельная инфраструктура для WebRTC). Скажи если нужно подключить.

⚠️ **Push-уведомления** — когда приложение закрыто, уведомление о сообщении не приходит. Делается через Firebase Cloud Messaging + Expo Notifications.

⚠️ **Картинки и файлы** — можно добавить через Firebase Storage.

⚠️ **История звонков** — пока показывается mock-данные, надо переписать на Firestore.

## Бесплатные лимиты Firebase

Для пет-проекта с парой десятков юзеров денег платить вообще не придётся:

- **Authentication**: 50,000 активных пользователей в месяц бесплатно
- **Firestore**: 50,000 чтений + 20,000 записей + 1 GB хранилища в день бесплатно
- **Cloud Messaging**: бесплатно безлимитно

Чтобы превысить — нужно реально активное приложение с тысячами пользователей.

---

## Если что-то не работает

**«Permission denied»** при отправке сообщения → проверь что Firestore включён в **test mode** (или что правила выше скопированы корректно)

**«FirebaseError: API key not valid»** → ты не заменил плейсхолдеры в `config.js` или скопировал не все 6 значений

**«auth/operation-not-allowed»** при регистрации → ты не включил Email/Password в Authentication (шаг 2)

**«Network request failed»** → нет интернета на устройстве

**После регистрации сразу выкидывает на экран входа** → нормально, это значит регистрация прошла, просто введи email/пароль и войди

## Как пересобрать APK с новым Firebase

После того как вставил ключи и проверил локально через Expo Go, можно пересобрать APK на EAS:

```bash
eas build --platform android --profile preview
```

Через 15 минут получишь обновлённый APK уже с подключённым Firebase.
